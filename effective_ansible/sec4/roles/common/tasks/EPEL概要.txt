[EPELリポジトリ]
業務用のサーバなどで利用するエンタープライズ向けLiunx用のソフトウェアを集積したリポジトリ。
CentOS等の公式パッケージに含まれないが有用なソフトウェアを提供している。
EPELはサードパーティ製のリポジトリに該当するため、EPELにあるパッケージを導入すると、そのパッケージはRed Hat社の
サポート対象外になる。
# yum install epel-release

・yum installの動作
(1)有効化されているリポジトリはどれかを確認する
(2)指定されたパッケージについて、依存関係を解決する（依存関係は、各rpmパッケージファイルに記載されている）
(3)対象ファイルをダウンロードする
(4)ダウンロードしたrpmファイルをインストールする

パッケージ ( package )
関連ファイルを一つにまとめたもの
httpdパッケージなら、/etc/httpd以下のファイルとか、/var/www/html以下のファイルとか。

リポジトリ ( repository )
パッケージをホストするサーバ
標準リポジトリと、サードパーティ製のリポジトリがある。
サードパーティ製のリポジトリは、EPEL(イーペル)リポジトリ、Remi(レミ)リポジトリなど。

RPMファイル
パッケージの形式のうちの一つ。Red Hat系OS用のパッケージがRPMパッケージ。Ubuntuだとdebパッケージ。
.rpmっていう拡張子になる

RPMデータベース
/var/lib/rpm/に存在する、rpm関連のデータを格納するデータベース。パッケージの情報を参照したりするのに利用する。
